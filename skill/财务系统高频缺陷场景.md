# 财务系统高频缺陷场景 Skill

本 Skill 用于指导 AI 在生成测试用例时，主动识别并覆盖财务系统中的高频缺陷场景，避免遗漏关键边界条件。

## 一、使用说明

当用户要求生成财务系统（报销单、付款单、预算变更、发票管理等）相关测试用例时，必须对照以下六类问题类型逐一检查，确保用例覆盖对应的测试要点。

## 二、常见问题类型与测试要点

### 1. 汇率 / 币种换算

**触发场景：** 涉及多币种、汇率转换、跨境付款的功能。

**测试要点：**
- 明确换算方向（本币→外币 / 外币→本币），不同方向分别出用例
- 明确不同币种的小数位数（如 JPY 无小数，USD 两位，BHD 三位）
- 覆盖汇率缺失 / 获取失败场景：系统需有兜底策略，并给出可验证的提示信息
- 验证换算结果在显示、计算、提交、落库、导出各环节一致

**典型 Bug 参考：** 授权商付款单手工发起提交时币种换算报错（后端汇率转换方向错误）；AI 数据未转换汇率直接提交。

### 2. 编辑 / 暂存后表单回显

**触发场景：** 任何支持编辑、暂存、复制、草稿的表单页面。

**测试要点：**
- 必测闭环：首次新建 → 保存 → 再次进入编辑 → 再次保存，验证前端展示与后端数据均已更新
- 暂存闭环：新增明细 → 暂存 → 退出 → 再进入，验证明细不丢失；反复暂存多次同样验证
- 复制入口：复制已有单据后进入编辑，验证所有字段正确回显
- 不同入口（新建 / 暂存 / 编辑 / 复制）需分别覆盖，不可合并

**典型 Bug 参考：** 海外公司税号配置表第二次打开未带出数据；员工报销单暂存后外勤单消失。

### 3. 预算系统 / 关联系统查询超时

**触发场景：** 涉及调用外部系统（预算系统、审批中心、采购系统等）的查询接口。

**测试要点：**
- 针对批量查询功能做小型性能测试：分别验证查询 10、50 个预算号（或其他关联对象）时响应是否满足要求
- 覆盖接口异常场景：超时 / 降级 / 部分成功，验证系统行为与提示是否符合预期
- 关联系统改动时，需同步验证本系统的配合改动是否生效

**典型 Bug 参考：** 查询预算系统接口超时，需支持传多个预算号批量查询。

### 4. 基础信息必填校验缺失

**触发场景：** 涉及收款信息（户名、账号、开户行、联行号、国家地区等）或其他关键基础信息的表单。

**测试要点：**
- 必填字段校验需前后端一致，分别验证前端拦截与后端接口拦截
- 统一梳理收款信息的必填项，确保校验逻辑复用，避免入口差异
- 覆盖所有入口：新建 / 暂存 / 编辑 / 复制，每个入口均需验证必填校验生效
- 改动某类明细表后，需做接口级回归，确保其他明细表不受影响

**典型 Bug 参考：** 福利费单据未校验户名不能为空（历史需求未要求，后补校验）；交通报销明细导入日期已填仍报错（后端改动误删其他明细取值逻辑）。

### 5. 金额精度问题

**触发场景：** 任何涉及金额计算、展示、提交的功能。

**测试要点：**
- 明确四舍五入规则及不同币种小数位数
- 覆盖全链路一致性：显示 → 计算 → 提交 → 落库 → 导出，各环节精度需一致
- 建立 JS 浮点精度典型用例集（如 0.1 + 0.2、大额金额累加等），专项回归
- 涉及与外部系统交互时，明确字段传值定义（如传 ID 还是描述文本）

**典型 Bug 参考：** 预算变更流程前端未处理数据精度；报销单维修类型传 ID 而非描述值给审批中心。

### 6. 流程中引用对象的异常链路

**触发场景：** 流程中引用了发票、合同、预算号、采购申请、供应商等外部对象。

**测试要点：**
- 覆盖引用对象的异常状态：被删除 / 状态变更（如已核销、已关闭）
- 覆盖并发场景：同一个对象被不同流程同时引用并操作
- 明确拦截策略：已使用 / 部分使用的对象不允许删除，需有明确提示
- 提交前验证引用对象的有效性，避免提交时才报错

**典型 Bug 参考：** 选中的海外发票删除后再提交单据，报错"发票唯一标识不存在"。

## 四、用例生成 Checklist

生成财务系统测试用例时，逐项确认：

- [ ] 是否涉及多币种/汇率？→ 覆盖换算方向、小数位、汇率缺失兜底
- [ ] 是否有编辑/暂存/复制入口？→ 覆盖保存后二次进入的闭环
- [ ] 是否调用外部系统？→ 覆盖超时/降级/部分成功 + 批量性能验证
- [ ] 是否有必填字段？→ 前后端一致，所有入口均覆盖
- [ ] 是否涉及金额计算？→ 覆盖显示/计算/提交/落库/导出全链路精度
- [ ] 是否引用了外部对象（发票/合同/预算/供应商）？→ 覆盖被删除/状态变更/并发操作异常链路
